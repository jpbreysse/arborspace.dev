Core Capabilities

Four pillars of intelligent knowledge

Every feature designed so humans navigate naturally and AI retrieves precisely.

📚

Living Knowledge Base

Create, organize, and evolve knowledge in a structured wiki. Hierarchical folders, typed documents (DOC, BUG, ROA), categories, tags, and virtual folders give every post six dimensions — serving both human navigation and machine intelligence.

Hierarchical folders Typed content Categories & tags Version history
🔍

Semantic Search

Find what you mean, not just what you typed. Posts are automatically split into chunks and embedded as 768-dimension vectors in pgvector. Hybrid search combines semantic similarity with full-text matching for precision results in under 100ms.

Vector embeddings pgvector Hybrid search Cosine similarity
🤖

AI-Powered RAG Chat

Ask questions in natural language. ArborSpace retrieves relevant chunks, injects them into editable prompt templates, and synthesizes answers with source citations. Consultants own the prompts — no developer needed to edit or create templates.

RAG pipeline Prompt templates Source citations SSE streaming
🏷️

Auto-Classification

AI classifies content into your domain taxonomy automatically. Every chunk inherits the post's category, tags, and permissions — search is filtered, results are access-controlled, and your taxonomy enriches relevance across the entire knowledge base.

Domain taxonomy AI classification RBAC enforcement Inherited permissions
Anatomy of a Post

One object — six dimensions

Every post serves both human navigation and machine intelligence. Searchable, classifiable, protectable.

🗂️

Category = Taxonomy

Hierarchical classification drives RAG search precision. Queries are filtered by technology branch for relevant results.

🏷️

Tags = Cross-cutting Labels

Flat labels that cut across categories. A post can be tagged CAPEX + Morocco + due-diligence. Powers faceted search.

📁

Virtual Folder = Navigation

Organizes posts for human browsing like a file system. A post can appear in multiple folders without duplication.

🔐

Groups = Who Can See It

Personal, public, or group-restricted. NDA-sensitive reports locked to specific teams. RBAC enforced at every level.

📊

Status = Lifecycle

Assigned / Closed tracks review workflow. Claim, assign, reopen. Full audit trail with complete version history.

📎

Source = Provenance

Tracks origin: manual creation, PDF import, SharePoint sync, workflow output. Know where every piece of knowledge came from.

Architecture

Three components. Zero external dependencies.

SvelteKit is the hub — it talks to PostgreSQL for data and Ollama for AI. They don't talk to each other. All on-premise via Docker Compose.

🐘

PostgreSQL 16

Posts, users, chunks, categories. pgvector for 768-dim embeddings. tsvector for full-text search. Cosine similarity + ts_rank.

Port 5432 · Persistent volume
⚡

SvelteKit

Wiki editor & content management. Hybrid search UI. RAG chat with SSE streaming. Auth, RBAC, admin panel. Document ingestion pipeline.

Node.js · Port 3000
🧠

Ollama

Mistral Nemo for generation & review. Nomic embedding model for vectorization. Streaming token generation. 100% local — no external API.

Port 11434 · ~4 GB models
PostgreSQL ←SQL/pgvector→ SvelteKit ←HTTP→ Ollama
🔒 On-premise — all three services on one server via Docker Compose. No external API calls. Data never leaves the network.
