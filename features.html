RAG — Retrieval-Augmented Generation

From question to sourced answer in 100ms

Documents are chunked, embedded, and stored as vectors. Hybrid search combines semantic similarity with keyword matching for precision results.

01
Chunk

Documents split into ~500 character chunks with 50 char overlap

→
02
Embed

nomic-embed-text-v2-moe encodes each chunk as 768-dim vector

→
03
Store

Vectors stored in PostgreSQL with pgvector for fast similarity search

→
04
Search

Hybrid: cosine similarity + ts_rank keyword matching, configurable weight

→
05
Generate

Mistral receives relevant chunks + prompt template, streams sourced answer

Hybrid Search in Action
// User asks a question
query: "O&M costs for solar in North Africa?"
// Step 1: Embed the question
vector = embed(query) → [0.032, -0.118, 0.245, ... 768 dims]
// Step 2: Hybrid search
semantic = cosine_similarity(vector, chunks) → top 5
keyword = ts_rank('solar & O&M & Africa') → top 5
combined = 0.7 × semantic + 0.3 × keyword
// Step 3: Generate with sources
answer = mistral(prompt_template + top_chunks)
// → "$4.3M annual, $28.5/kW/year [source: Noor III report]"

Semantic + keyword, combined

"Solar PV maintenance costs" finds "O&M budget for photovoltaic installations" — semantic search understands meaning, not just words. Keyword search catches exact technical terms that embeddings might miss.

Configurable weight lets you tune the balance: more semantic for exploratory questions, more keyword for exact technical lookups.

Knowledge Graph

See how your knowledge connects

Visual representation of document connections — discover relationships that folder structures hide.

Knowledge Graph Visualization
━ Semantic similarity ╌ Shared tags

Two types of connections

Semantic edges — Posts linked by embedding similarity when above a configurable threshold (default: 65%). Discovers relationships that humans wouldn't manually tag.

Tag edges — Posts linked by shared tags using Jaccard similarity. Explicit connections based on your classification choices.

Interactive exploration

Click nodes to navigate directly to posts. Zoom and pan across your knowledge landscape. Filter by category and adjust the similarity threshold to reveal stronger or weaker connections.

Workflows

Multi-step LLM pipelines — defined as wiki posts

Chain multiple AI steps with variable interpolation. Each step can inject RAG context. Results saved as new posts. No code required.

Write workflows in markdown

Define workflows directly in the wiki editor using step markers. Each step's output becomes a variable available to subsequent steps — {{step1}} feeds into Step 2, and so on.

RAG context is injected at each step, so every stage of the pipeline has access to your full knowledge base. Results stream in real-time and are automatically saved as new posts.

Sequential execution Variable interpolation RAG at each step Auto-save results
Workflow Definition
## Step 1: Summarize
Summarize the document: {{input}}
## Step 2: Extract Entities
Extract key entities from: {{step1}}
## Step 3: Final Report
Create structured report from {{step1}} and {{step2}}
✦ Result saved as post → immediately searchable & vectorized
Document Enrichment

AI classifies and enriches every document

Automatic classification via LLM — every document gets a doc type, quality score, summary, and topic extraction without manual effort.

📋

Doc Type Classification

AI determines if a post is a technical document, meeting notes, report, correspondence, tutorial, or other type. Classification drives search filtering and template matching.

technicaldoc meeting report correspondence
⭐

Quality Scoring

Content quality rated by AI based on completeness, structure, clarity, and sourcing. Helps knowledge managers prioritize curation efforts and surface the best content first.

📝

Auto-Summary

AI generates a concise summary for every document on save. Summaries appear in search results and folder views, helping users find the right content without opening every post.

🔖

Topic Extraction

Key topics automatically extracted and stored as structured metadata. Powers topic-based navigation, trend analysis, and cross-document discovery across your knowledge base.

Permissions & Groups

Share the knowledge, protect the access

Group-based access control enforced at every level — from folder browsing to search results to RAG chat answers.

👤

User Roles

Admin and User roles with distinct capabilities. Admins manage groups, configure taxonomy, and control platform settings. Users create, edit, and consume knowledge.

👥

Groups

Create groups and add members. Groups map to organizational units, project teams, or access tiers. A user can belong to multiple groups simultaneously.

👁️

Three Visibility Levels

Public — anyone can view. Private — only the author. Groups — only members of specified groups. Enforced at query time across search, browse, and chat.

🔑

Read/Write Permissions

Granular access control beyond visibility. Control who can edit, comment, assign, and manage posts independently of who can read them. Perfect for regulated environments.
